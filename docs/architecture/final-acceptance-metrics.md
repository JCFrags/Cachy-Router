# Final Acceptance Metrics

This is the durable acceptance matrix for the 90-100% Cachy Router target.
It converts the final-goal bullet list into a verifiable project contract.

The build is not considered final while any metric is marked `partial`,
`planned`, `live-gated`, or `blocked-by-upstream`. A status of `done` means the
repo currently contains direct implementation evidence or an offline check for
that metric. A status of `live-gated` means the code path exists but requires
operator-supplied live workers before it can be proven for a deployment.

Run:

```bash
python3 scripts/validate_acceptance_metrics.py --json
python3 scripts/validate_acceptance_metrics.py --strict --json
python3 scripts/cache_router_release_gap_report.py --summary
```

The first command validates this document's structure. The strict command is
the final-release gate: it must fail until every metric is actually done.
The release-gap report validates that every unfinished row maps to a concrete
live, duration, clean-checkout, or upstream-runtime proof bucket.

## Core API

- [live-gated] OpenAI `/v1/models` succeeds through the router with HTTP 200 on every healthy deployment -- evidence: `scripts/cache_router_daemon.py` implements `/v1/models`; `scripts/cache_router_live_endpoint_matrix.py` provides an operator-run live matrix gate that requires `/v1/models` to return HTTP 200 and list every route-ready configured model; deployment evidence still required.
- [live-gated] `/v1/completions` pass-through succeeds for every configured worker model -- evidence: `scripts/cache_router_daemon.py` proxies normal completions; `scripts/cache_router_live_endpoint_matrix.py` probes one plain completion per configured model and one fallback-disabled `cache_router.mode=bypass` completion per worker/model pair, asserting HTTP 200, non-empty choices, request/trace headers, and selected-worker headers; deployment evidence still required.
- [live-gated] `/v1/chat/completions` pass-through succeeds for every configured worker model -- evidence: `scripts/cache_router_daemon.py` proxies normal chat completions; `scripts/cache_router_live_endpoint_matrix.py` probes one plain chat completion per configured model and one fallback-disabled `cache_router.mode=bypass` chat completion per worker/model pair, asserting HTTP 200, message content, request/trace headers, and selected-worker headers; deployment evidence still required.
- [done] Router strips `cache_router` before forwarding bypass requests to `llama-server` -- evidence: `scripts/cache_router_daemon.py` removes the extension in bypass mode.
- [done] Streaming pass-through works for normal non-cached and bypass requests -- evidence: `scripts/cache_router_daemon.py` routes normal `stream=true` OpenAI requests through a streaming branch that relays backend bytes with `read1()` and omits `Content-Length`; `cache_router.mode=bypass` is parsed before the cached-mode stream guard, strips the extension, and uses the same proxy path. `scripts/cache_router_daemon_smoke_test.py` verifies loopback completion and chat event-stream responses expose the first event line, preserve `text/event-stream`, forward the `[DONE]` terminator, and still keep cached build/use streaming rejection separate.
- [done] Cached build/use requests reject streaming clearly until cached streaming is implemented -- evidence: `scripts/cache_router_daemon.py` returns a JSON error when cached build/use/auto/refresh mode has `stream=true`; `scripts/cache_router_daemon_smoke_test.py` verifies the request is not forwarded to workers.
- [done] Every router response includes selected-worker debug metadata in headers or decision logs -- evidence: routed proxy and streaming responses include router-owned `X-Cache-Router-Request-ID`, `X-Cache-Router-Trace-ID`, and selected `X-Cache-Router-Worker`; successful cache-extension responses include the same IDs, selected worker, and cache hit level for cache-use responses; router-generated health, metrics, discovery, auth, invalid-JSON, disabled-admin, and pre-routing rejection responses use the same request/trace headers with `X-Cache-Router-Worker: none`; routed failure paths emit request-correlated decision events when a worker attempt exists. `scripts/cache_router_daemon.py` centralizes default debug headers in `send_json()` and `send_text()`, while preserving more specific routed/cache headers, and `scripts/cache_router_daemon_smoke_test.py` verifies the endpoint-family matrix offline.
- [done] Router returns OpenAI-shaped errors for unavailable models or workers -- evidence: `scripts/cache_router_daemon.py` maps unknown models to `model_not_found`, unknown workers to `worker_not_found`, and no ready workers to `service_unavailable`; `scripts/cache_router_daemon_smoke_test.py` verifies all three offline.
- [done] `/health` distinguishes router process health from worker readiness summary -- evidence: `/health` reports router bind/pid plus worker health and ready counts when authorized or unauthenticated.
- [done] `/router/workers` reports configured workers and current states -- evidence: `worker_summaries()` backs `/router/workers` with health, model-readiness, route-readiness, availability, transport, and model/runtime fields.

- [done] Recovered workers re-enter candidate set only after readiness passes -- evidence: the daemon smoke test flips worker `/v1/models` from loading 503 to HTTP 200 with the configured model and verifies the worker is selected again.
- [done] Sidecar readiness is tracked separately from model readiness -- evidence: `scripts/cache_router_daemon.py` keeps route readiness defined as worker `/health` plus worker `/v1/models`, exposes `sidecar_readiness` separately in worker summaries, exports `cachy_router_worker_sidecar_ready`, and does not probe sidecars during normal pass-through candidate selection; `scripts/cache_router_daemon_smoke_test.py` verifies local transport `not_applicable`, HTTP sidecar ready/down/worker-mismatch states, and normal `/v1/completions` still routes for a model-ready HTTP-transport worker while its sidecar is unhealthy. - [done] Router keeps serving healthy workers while one worker is down -- evidence: `scripts/cache_router_daemon_smoke_test.py` uses two loopback workers and verifies a model-loading first worker is skipped while a ready backup serves traffic.
- [done] Router returns a clear 503 only when no valid worker exists -- evidence: `scripts/cache_router_daemon_smoke_test.py` makes every fake worker not ready and verifies `/v1/models` plus `/v1/completions` return `503 service_unavailable`.
- [done] Setup doctor catches unreachable router, worker, and sidecar endpoints -- evidence: `scripts/cache_router_setup_doctor_matrix_test.py` runs loopback live checks and separately asserts closed router, worker, and sidecar endpoints produce the expected failure codes.

- [done] Idle healthy worker is preferred over busy healthy worker for normal traffic -- evidence: `rank_worker()` sorts lower busy scores first; `scripts/cache_router_daemon_smoke_test.py` marks the first fake worker slot busy and verifies normal traffic selects the idle backup.
- [done] Explicit `worker_id` target routes to that worker when healthy -- evidence: request extension `worker_id` feeds worker selection; `scripts/cache_router_daemon_smoke_test.py` verifies `worker_id=worker-backup` with `allow_fallback=false` selects only the backup worker.
- [done] Explicit `worker_id` target fails when `allow_fallback=false` and worker is unavailable -- evidence: ordered candidates exclude fallback workers when fallback is false; `scripts/cache_router_daemon_smoke_test.py` verifies an unavailable explicit target returns `503 service_unavailable` without hitting the backup worker.
- [done] Fallback routes to another compatible worker when `allow_fallback=true` -- evidence: ordered candidates include other workers and cached use iterates candidates; `scripts/cache_router_daemon_smoke_test.py` verifies a preferred worker returning loading 503 falls back to a ready backup with bounded `worker_capacity` reason.
- [done] Scheduler logs every candidate score input -- evidence: `scripts/cache_router_daemon.py` records a bounded `scheduler` object on routing decision events with candidate worker ID, eligibility, readiness state, availability reason, busy score, active request count, hot-residency flag, preferred-worker flag, order index, and rank tuple; `schemas/cache-router/cache-decision-event.schema.json` permits only that redacted scheduler shape; `scripts/cache_router_daemon_smoke_test.py` verifies the trace includes both loopback workers for normal proxy decisions.
- [done] Scheduler records why the winning worker won -- evidence: `scripts/cache_router_daemon.py` stores `winner_worker_id`, `winner_rank`, and a bounded `winner_reason` in the decision-event scheduler trace for normal proxy, cache build, and cache use selections; `scripts/cache_router_daemon_smoke_test.py` verifies the winner and reason are present for proxied traffic.
- [done] Queue depth affects routing decisions -- evidence: `scripts/cache_router_daemon.py` tracks opt-in per-worker router queue depth, includes `queue_depth` in the normal scheduler rank tuple after busy score and active requests, records `queue_depth` and `queue_capacity` in scheduler candidate traces, and `scripts/cache_router_daemon_smoke_test.py` seeds a higher queue depth on the previously preferred worker and verifies the lower-queue backup wins with schema-valid decision evidence. - [done] Running request count affects routing decisions -- evidence: `scripts/cache_router_daemon.py` tracks per-worker in-flight backend attempts and includes `active_requests` in the worker rank tuple; `scripts/cache_router_daemon_smoke_test.py` holds one fake worker request open, resets cold tie-breaking to favor that worker, and verifies a second request selects the other ready worker because the first worker has `active_requests=1`.
- [done] Cache locality affects routing decisions -- evidence: `scripts/cache_router_daemon.py` passes worker residency into cached-use candidate ordering, and `scripts/cache_router_daemon_smoke_test.py` verifies a hot local cached use selects the resident worker, exposes `X-Cache-Router-Cache-Hit-Level: local_nvme`, performs no hydration, and emits a correlated decision event for the selected worker. - [done] Saturation can override cache locality when a hot worker has `busy_score>=2` and a compatible idle worker is available -- evidence: `rank_worker()` sorts `busy_score` and active requests before hot residency, normal routing smoke verifies busy-score routing away from a busy worker, and cached-use smoke seeds a hot local cache on a busy worker, verifies the idle compatible worker wins, hydrates the durable blob, restores there, and records scheduler inputs showing hot/busy main lost to cold/idle backup. - [done] Cold requests distribute across workers instead of pinning one worker forever -- evidence: `scripts/cache_router_daemon.py` rotates equal-score cold candidate ordering when there is no explicit worker target and no hot-residency preference; `scripts/cache_router_daemon_smoke_test.py` sends equal-score normal completions and verifies both loopback workers receive traffic.
- [done] No worker receives normal traffic after it is marked model-incompatible, and cached restore skips strict-key-incompatible workers -- evidence: `scripts/cache_router_daemon.py` filters normal candidates by requested model and readiness before forwarding, and checks `worker_cache_compatibility_mismatch()` before cached hydrate, restore, or generation; `scripts/cache_router_daemon_smoke_test.py` adds a dedicated `worker-incompatible` fake backend, verifies `/router/workers` reports `model_mismatch`, verifies normal scheduler candidates exclude that worker for `fake-model`, verifies explicit fallback-disabled routing to it returns no selected worker, verifies its backend receives no forwarded completion, and verifies strict cached-use negatives perform no restore or generation. ## Correctness
